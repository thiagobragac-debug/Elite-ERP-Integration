# ESLint Configuration Guide

This document explains the ESLint rules configured for the Tauze ERP project.

## Overview

ESLint is configured with comprehensive rules for:
- **React & React Hooks**: Best practices for React components and hooks
- **TypeScript**: Type safety and modern TypeScript patterns
- **Accessibility (a11y)**: WCAG compliance and accessibility best practices
- **Performance**: Catching common performance anti-patterns
- **Code Quality**: Maintainability and bug prevention

## Installed Plugins

- `eslint-plugin-react`: React-specific linting rules
- `eslint-plugin-react-hooks`: Enforces Rules of Hooks
- `eslint-plugin-react-refresh`: Fast Refresh compatibility
- `eslint-plugin-jsx-a11y`: Accessibility rules for JSX elements
- `typescript-eslint`: TypeScript-specific linting

## Rule Categories

### React Rules

**Enforced (Errors):**
- `react-hooks/rules-of-hooks`: Ensures Hooks are called in the correct order
- `react/jsx-key`: Requires `key` prop for elements in arrays
- `react/jsx-no-target-blank`: Prevents security issues with `target="_blank"`
- `react/no-children-prop`: Prevents passing `children` as a prop
- `react/no-deprecated`: Flags deprecated React APIs

**Warnings:**
- `react-hooks/exhaustive-deps`: Suggests missing dependencies in useEffect/useCallback
- `react/no-array-index-key`: Warns against using array index as key
- `react/no-danger`: Warns about using dangerouslySetInnerHTML
- `react/self-closing-comp`: Suggests self-closing for components without children

### Accessibility Rules (jsx-a11y)

**Enforced (Errors):**
- `alt-text`: Requires alt text for images
- `aria-*`: Validates ARIA attributes and roles
- `heading-has-content`: Ensures headings have content
- `html-has-lang`: Requires lang attribute on html element

**Warnings:**
- `click-events-have-key-events`: Suggests keyboard handlers for click events
- `label-has-associated-control`: Ensures form labels are properly associated
- `no-autofocus`: Warns against autofocus attribute
- `tabindex-no-positive`: Discourages positive tabindex values

### TypeScript Rules

**Enforced:**
- `@typescript-eslint/no-unused-vars`: Flags unused variables (with `_` prefix exception)
- `@typescript-eslint/no-explicit-any`: Warns about using `any` type

**Best Practices:**
- Prefix unused parameters with `_` to suppress warnings
- Use proper TypeScript types instead of `any`
- Leverage optional chaining (`?.`) for safer property access

### Performance & Code Quality

**Complexity Limits:**
- `complexity`: Maximum cyclomatic complexity of 20
- `max-depth`: Maximum nesting depth of 4
- `max-lines`: Maximum 500 lines per file (excluding blanks/comments)
- `max-params`: Maximum 5 parameters per function

**Best Practices:**
- `no-console`: Only `console.warn` and `console.error` allowed
- `prefer-const`: Use `const` for variables that are never reassigned
- `eqeqeq`: Always use `===` instead of `==`
- `no-await-in-loop`: Warns about awaiting in loops (performance)

### Bug Prevention

**Enforced Errors:**
- `array-callback-return`: Ensures array methods return values
- `no-constructor-return`: Prevents returning values from constructors
- `no-promise-executor-return`: Prevents returning in Promise executor
- `no-self-compare`: Catches `x === x` comparisons
- `require-atomic-updates`: Prevents race conditions with async updates

## Running ESLint

### Check for Issues
```bash
npm run lint
```

### Auto-fix Issues
```bash
npm run lint:fix
```

### Check Specific Files
```bash
npx eslint src/components/MyComponent.tsx
```

### Fix Specific Files
```bash
npx eslint src/components/MyComponent.tsx --fix
```

## CI/CD Integration

ESLint runs automatically in the CI pipeline:
1. On every pull request
2. Blocks merge if linting violations exist (as per Requirement 12.6)

## Working with Prettier

ESLint is configured to work seamlessly with Prettier:
- Prettier handles code formatting (spacing, line breaks, etc.)
- ESLint handles code quality and potential bugs
- Run `npm run format` for Prettier formatting
- Run `npm run lint:fix` for ESLint fixes

## Common Issues & Solutions

### Issue: "Expected { after 'if' condition"
**Solution:** Always use curly braces for if statements, even one-liners
```typescript
// ❌ Bad
if (condition) return;

// ✅ Good
if (condition) {
  return;
}
```

### Issue: "React Hook useEffect has missing dependencies"
**Solution:** Add dependencies or use useCallback/useMemo
```typescript
// ❌ Bad
useEffect(() => {
  doSomething(value);
}, []); // 'value' is missing

// ✅ Good
useEffect(() => {
  doSomething(value);
}, [value]);
```

### Issue: "A form label must be associated with a control"
**Solution:** Use htmlFor attribute or nest input inside label
```typescript
// ❌ Bad
<label>Name</label>
<input />

// ✅ Good
<label htmlFor="name">Name</label>
<input id="name" />

// ✅ Also Good
<label>
  Name
  <input />
</label>
```

### Issue: "Visible, non-interactive elements with click handlers must have keyboard listeners"
**Solution:** Add onKeyPress handler or use button element
```typescript
// ❌ Bad
<div onClick={handleClick}>Click me</div>

// ✅ Good
<div 
  onClick={handleClick}
  onKeyPress={handleClick}
  role="button"
  tabIndex={0}
>
  Click me
</div>

// ✅ Better
<button onClick={handleClick}>Click me</button>
```

## Customizing Rules

To adjust rule severity, edit `eslint.config.js`:
- `'error'`: Fails lint check
- `'warn'`: Shows warning but doesn't fail
- `'off'`: Disables the rule

Example:
```javascript
rules: {
  'no-console': 'off', // Allow all console statements
  'complexity': ['warn', 30], // Increase complexity limit to 30
}
```
